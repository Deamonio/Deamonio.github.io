1. OVERVIEW: THE ID BECOMES A TECHNICAL DECLARATION

Deamon's Project ID is not just a simple identification code, but a technical declaration that reflects the philosophy of the 'Unseen Connection'. This system is designed to engrave 'Unwavering Reliability' and 'Responsible Precision' into the code's signature.

ID Structure Formula (Total 14‚Äì16 Digits)

$$[ID] = [P] \ [YYMM] \ [EAWDC] \ [X] \ [Project Code] \ [Checksum]$$

2. DETAILED ID COMPONENT MANUAL

A. Prefix (P) - 1 Digit (Required)

This reflects Deamon's 'Function for Good' philosophy by classifying the ultimate goal of the project.

Code Type Meaning Example Project
G **Good** Public/Rescue solutions for a better life and sustainable society. Disaster rescue robot, energy efficiency optimization system.
**S** **Solution** Custom innovation solutions and commercial services for general clients. Smart factory, general B2B/B2C services.
**R** **Research** Internal R&D, pure technology experiments, exploration for creative/artistic expression. Horror TV experiment, AI interaction method research.

B. Time Stamp (YYMM) - 4 Digits (Required)

Indicates the year and month the project was officially started or registered. (E.g., Oct 25 $\rightarrow$ 2510)

C. Fusion Type (EAWDC) - 5 Digits (Required)

Indicates the technological areas used from Deamon's 5 core architectures, marking the depth of the 'Unseen Connection'. Enter the corresponding code if used, or 0 (Zero) if not used.

Position Code Technology Area If Used If Not Used
1 E Embedded (Control, Firmware, Sensor) **E** **0**
2 A AI (Cognition, Intelligence, Analysis, Modeling) **A** **0**
3 W Web (Interface, Front-end, UX) **W** **0**
4 D Data (DB, Storage, Pipeline, Management) **D** **0**
5 C Communication (Network, API, Backend Server) **C** **0**

Example: For AI analysis backend only: 0A0DC

D. Specialty Code (X) - 1 Digit (Required/Special)

A reserved field for special or future technologies not covered by EAWDC.

Code Type Meaning
**0** None No special technology used (Most projects)
**B** Blockchain Distributed ledger technology, NFT, Smart Contract
**Q** Quantum Quantum computing or quantum security technology
**H** Hardware Specialized hardware/ASIC design

E. Project Code - 3 Digits (Required)

A unique sequential number assigned within each Prefix (G, S, R). Starts from 001.

F. Checksum - 1‚Äì3 Digits (Required)

A verification code to ensure 'Unwavering Reliability'. It validates for typos or errors in the ID, calculated using the Modulus 9 method, etc.

3. ID GENERATION AND VERIFICATION PROCEDURE

Step 1: Project Nature and Technology Confirmation

  • Prefix (P): Select one from G, S, or R
  • Fusion Type (EAWDCX): Confirm the 6-digit code based on the technology stack used.
  • YYMM / Project Code: Enter the confirmed date and sequence number.

Step 2: Checksum Calculation

  1. Convert all characters of the ID to numbers according to predefined rules. (G=7, S=19, E=5, A=1, W=23, D=4, C=3)
  2. Sum all converted numbers to find the Total Sum.
  3. Confirm the Checksum value by taking the remainder when Total Sum is divided by 9.

Step 3: Final ID Completion

  • The calculated Checksum value is finally appended to the end of the ID.
Example: Disaster Rescue Robot **G 2406 EAWDC 0 005 8**
**Analysis** **G**ood project starting June 2024. Uses all core technologies (EAWDC), no special technology (X), 5th sequence number. Verification code **8**.

4. ID MAINTENANCE AND MANAGEMENT

  • Exclusive Sequence: Each Prefix (G, S, R) must manage a separate sequence number (Project Code).
  • Fusion Type Modification: If the technology stack changes during the project, the ID must be re-issued reflecting the **latest technology stack**.
  • Checksum Validity: When entering the ID into a database or system, validity must be checked using the Checksum to maintain **data integrity**.
